# TypeScript

这是一份动态维护的TS开发与风格规范。

https://github.com/google/ts-style

https://github.com/Microsoft/TypeScript/wiki/Coding-guidelines

### 命名

| 优先级 | 权重 | 内容                                                         |
| ------ | ---- | ------------------------------------------------------------ |
|        |      | 类型名用Pascal命名法                                         |
|        |      | interface名字用Pascal命名法，且前面不需要加```I```           |
|        |      | Enum类型也用Pacal命名法                                      |
|        |      | 函数名使用camel命名法                                        |
|        |      | 本地变量以及属性名用camel命名法                              |
|        |      | 私有属性以及变量不用```_``` 或者 ```$```开头，因为可能会和VueJS本身冲突。请使用```m_```开头 |
|        |      | 变量名使用全称而不是简称                                     |

### 组件

| 优先级 | 权重 | 内容                                                         |
| ------ | ---- | ------------------------------------------------------------ |
|        |      | 一个文件是一个逻辑组件，例如```scanner.ts```,```types.ts```等 |
|        |      | 添加或者修改全局组件需要告知每一个人                         |

### 类型

| 优先级 | 权重 | 内容                                                         |
| ------ | ---- | ------------------------------------------------------------ |
|        |      | 不要随便导出类型，除非这个类型需要在多个模块之间共享         |
|        |      | 不要给全局空间引入类型，如果必须引入的话，需要开会讨论，并告知每一个人 |
|        |      | 全局共享的类型，需要集中定义在```types.ts```类似的这样一个类型定义配置文件中 |
|        |      | 在一个文件内，类型定义一定需要写在最开始的地方               |
|        |      | 尽量使用undefine而不是null                                   |
